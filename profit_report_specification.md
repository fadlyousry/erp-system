# 📊 تقرير الأرباح المفصّل — مواصفات التصميم

## الهدف
بناء تقرير أرباح شامل ودقيق يحسب **الربح الفعلي لكل صنف** بناءً على سعر البيع المحدد في الفاتورة (وليس السعر العام فقط)، مع إمكانية تحديد الفترة الزمنية.

---

## 📐 مصادر البيانات (من الـ Schema)

### ١. بيانات المبيعات
| الجدول | الحقل | الاستخدام |
|--------|-------|-----------|
| `Sale` | `id`, `total`, `discount`, `invoiceDate`, `customerId`, `saleType` | بيانات الفاتورة الأساسية |
| `SaleItem` | `price`, `quantity`, `discount`, `variantId` | **سعر البيع الفعلي لكل صنف** (هذا هو المفتاح — السعر المخصص للعميل) |
| `Variant` | `cost`, `price`, `productId` | **سعر التكلفة** للصنف + السعر الأساسي |
| `Product` | `name`, `cost`, `basePrice`, `categoryId` | اسم المنتج + تكلفة المنتج الأساسية |
| `Customer` | `name` | اسم العميل |

### ٢. بيانات المرتجعات
| الجدول | الحقل | الاستخدام |
|--------|-------|-----------|
| `Return` | `id`, `total`, `saleId`, `createdAt` | بيانات المرتجع |
| `ReturnItem` | `price`, `quantity`, `variantId` | سعر وكمية الصنف المرتجع |

### ٣. بيانات المشتريات (لتحديد التكلفة الفعلية)
| الجدول | الحقل | الاستخدام |
|--------|-------|-----------|
| `Purchase` | `id`, `total`, `expensesTotal`, `isExpensesDistributed` | تكلفة المشتريات + المصاريف الإضافية |
| `PurchaseItem` | `cost`, `quantity`, `variantId` | **سعر شراء كل صنف** (التكلفة الفعلية) |

### ٤. بيانات المصروفات
| الجدول | الحقل | الاستخدام |
|--------|-------|-----------|
| `Expense` | `amount`, `title`, `expenseDate`, `categoryId` | المصروفات التشغيلية |
| `ExpenseCategory` | `name` | تصنيف المصروف |

## 🔢 طريقة حساب الربح (الأساس)

### الربح على مستوى الصنف (Item-Level Profit)

```
ربح الصنف = (سعر البيع في الفاتورة × الكمية) - (خصم الصنف) - (تكلفة الصنف × الكمية)
```

#### أولوية تحديد التكلفة:
1. **`Variant.cost`** — تكلفة الصنف (المخزنة في الـ Variant) ← **الأولوية الأولى**
2. **`Product.cost`** — تكلفة المنتج الأساسية ← **بديل إذا لم توجد تكلفة الـ Variant**
3. **`0`** — إذا لم توجد أي تكلفة (مع تحذير في التقرير)

> [!IMPORTANT]
> **السعر المستخدم هو `SaleItem.price` وليس `Variant.price` أو `Product.basePrice`**
> لأن العميل قد يحصل على سعر مخصص مختلف عن السعر الأساسي.

### صيغة الحساب التفصيلية لكل فاتورة:

```
إيراد الفاتورة = Σ (SaleItem.price × SaleItem.quantity) - Σ (SaleItem.discount)
تكلفة الفاتورة = Σ (Variant.cost × SaleItem.quantity)
خصم الفاتورة الإجمالي = Sale.discount
إجمالي ربح الفاتورة = إيراد الفاتورة - تكلفة الفاتورة - خصم الفاتورة الإجمالي
هامش الربح % = (إجمالي ربح الفاتورة / إيراد الفاتورة) × ١٠٠
```

### حساب تأثير المرتجعات:

```
خسارة المرتجع = Σ (ReturnItem.price × ReturnItem.quantity) - Σ (Variant.cost × ReturnItem.quantity)
// أي: الربح الذي "خسرناه" بسبب المرتجع
```

## 📑 أقسام التقرير

### القسم ١: الملخص العام (KPI Cards)
| البطاقة | الحساب |
|---------|--------|
| 💰 إجمالي المبيعات | `Σ Sale.total` (في الفترة) |
| 📦 تكلفة البضاعة المباعة (COGS) | `Σ (Variant.cost × SaleItem.quantity)` لكل فاتورة |
| 📈 إجمالي الربح (Gross Profit) | إجمالي المبيعات - COGS - خصومات |
| 🔄 المرتجعات | `Σ Return.total` |
| 📈 صافي الربح بعد المرتجعات | إجمالي الربح - ربح المرتجعات المفقود |
| 💸 المصروفات التشغيلية | `Σ Expense.amount` (في الفترة) |
| ✅ **صافي الربح النهائي (Net Profit)** | صافي الربح بعد المرتجعات - المصروفات |
| 📊 هامش الربح الإجمالي % | (إجمالي الربح / إجمالي المبيعات) × ١٠٠ |
| 📊 هامش الربح الصافي % | (صافي الربح النهائي / إجمالي المبيعات) × ١٠٠ |
| 🔢 عدد الفواتير | عدد الفواتير في الفترة |
| 📋 متوسط ربح الفاتورة | صافي الربح / عدد الفواتير |

### القسم ٢: تفاصيل الربح حسب الفاتورة
جدول يعرض كل فاتورة مع:

| العمود | المصدر |
|--------|--------|
| رقم الفاتورة | `Sale.id` |
| التاريخ | `Sale.invoiceDate` |
| العميل | `Customer.name` |
| نوع البيع | `Sale.saleType` |
| إجمالي الفاتورة | `Sale.total` |
| التكلفة | `Σ (Variant.cost × quantity)` per sale |
| الخصم | `Sale.discount` |
| الربح | الإجمالي - التكلفة - الخصم |
| هامش الربح % | (الربح / الإجمالي) × ١٠٠ |
| عدد الأصناف | عدد `SaleItem` records |

### القسم ٣: تفاصيل الربح حسب المنتج (أعلى المنتجات ربحية)
جدول يعرض كل منتج/صنف مع:

| العمود | المصدر |
|--------|--------|
| اسم المنتج | `Product.name` |
| اللون/المقاس | `Variant.color` + `Variant.productSize` |
| الكمية المباعة | `Σ SaleItem.quantity` |
| متوسط سعر البيع | `avg(SaleItem.price)` |
| سعر التكلفة | `Variant.cost` |
| إجمالي الإيراد | `Σ (price × quantity)` |
| إجمالي التكلفة | `Σ (cost × quantity)` |
| الربح | الإيراد - التكلفة |
| هامش الربح % | (الربح / الإيراد) × ١٠٠ |

> مرتب تنازلياً حسب الربح

### القسم ٤: تفاصيل الربح حسب التصنيف (Category)
| العمود | المصدر |
|--------|--------|
| التصنيف | `Category.name` |
| عدد المنتجات المباعة | عدد المنتجات الفريدة |
| الكمية الإجمالية | `Σ quantity` |
| إجمالي الإيراد | `Σ (price × quantity)` |
| إجمالي التكلفة | `Σ (cost × quantity)` |
| الربح | الإيراد - التكلفة |
| هامش الربح % | النسبة المئوية |

### القسم ٥: المصروفات التشغيلية
| العمود | المصدر |
|--------|--------|
| التصنيف | `ExpenseCategory.name` |
| عدد المصروفات | `count` |
| الإجمالي | `Σ Expense.amount` |
| النسبة من الإيراد | (الإجمالي / إجمالي المبيعات) × ١٠٠ |

### القسم ٦: تحليل المرتجعات
| العمود | المصدر |
|--------|--------|
| رقم المرتجع | `Return.id` |
| فاتورة الأصل | `Return.saleId` |
| العميل | `Customer.name` |
| المبلغ | `Return.total` |
| الربح المفقود | المبلغ - التكلفة |

### القسم ٧: تحليل الربح حسب العميل (اختياري)
| العمود | المصدر |
|--------|--------|
| اسم العميل | `Customer.name` |
| عدد الفواتير | count |
| إجمالي المشتريات | `Σ Sale.total` |
| إجمالي الربح | `Σ profit` |
| هامش الربح % | النسبة |

## ⚠️ حالات خاصة (Edge Cases)

### ١. منتجات بدون تكلفة
- إذا `Variant.cost` = `null` أو `0` و `Product.cost` = `null`
- **الحل:** يظهر تحذير ⚠️ بجانب المنتج ويُحسب الربح = إيراد البيع بالكامل (مع ملاحظة "تكلفة غير محددة")
- يظهر في الملخص: "عدد المنتجات بدون تكلفة محددة: X"

### ٢. خصم على مستوى الصنف + خصم على مستوى الفاتورة
- **خصم الصنف:** `SaleItem.discount` يُخصم من ربح الصنف مباشرة
- **خصم الفاتورة:** `Sale.discount` يُخصم من إجمالي ربح الفاتورة
- **صيغة الصنف الكاملة:**
```
ربح الصنف = (SaleItem.price × SaleItem.quantity) - SaleItem.discount - (Variant.cost × SaleItem.quantity)
ربح الفاتورة = Σ ربح الأصناف - Sale.discount
```

### ٣. سعر مخصص للعميل (السيناريو الأساسي)
- **مثال:** سعر المنتج الأساسي = ١٠٠ جنيه، لكن العميل اشترى بـ ٨٠ جنيه
- `SaleItem.price` = ٨٠ (السعر الفعلي في الفاتورة)
- `Variant.cost` = ٦٠ (تكلفة الشراء)
- **الربح الصحيح** = ٨٠ - ٦٠ = **٢٠ جنيه** (وليس ١٠٠ - ٦٠ = ٤٠)

### ٤. بيع بأقل من التكلفة (خسارة)
- **مثال:** `SaleItem.price` = ٤٠، `Variant.cost` = ٦٠
- **الربح** = ٤٠ - ٦٠ = **-٢٠** (خسارة)
- يظهر بالأحمر مع علامة ⬇️

### ٥. المرتجعات الجزئية
- إذا تم إرجاع ٢ من ٥ قطع
- الربح المفقود = (سعر البيع - التكلفة) × الكمية المرتجعة فقط

## 🏗️ خطة التنفيذ

### المرحلة ١: Backend (API)
1. إنشاء دالة `getProfitReport(params)` في `db-service.js`
2. استعلام المبيعات مع `SaleItems` + `Variants` + `Products` في الفترة
3. استعلام المرتجعات مع `ReturnItems` + `Variants`
4. استعلام المصروفات في نفس الفترة
5. حساب الأرباح على مستوى كل صنف ثم تجميعها

### المرحلة ٢: IPC Bridge
1. تسجيل `db:getProfitReport` في `database-ipc-handlers.js`
2. تسجيل الـ API في `preload.js`

### المرحلة ٣: Frontend (UI)
1. إضافة تاب "الأرباح" أو صفحة مستقلة
2. واجهة الفلاتر (تاريخ من/إلى + فلاتر اختيارية)
3. عرض KPI Cards + الجداول التفصيلية
4. زر طباعة التقرير
